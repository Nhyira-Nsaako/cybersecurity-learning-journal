# Information Security Fundamentals

## Overview
Information Security (InfoSec) focuses on protecting systems, networks, and data from unauthorized access, misuse, or damage. It involves identifying risks, preventing attacks, detecting threats, responding to incidents, and continuously improving security posture.


## Key Definitions

### Risk
Risk is the potential for a malicious event to occur that could negatively impact an organisation or system.

> In simple terms: Risk = what could go wrong + how bad it would be


### Threat
A threat is any potential cause of an event that could result in harm to a system or organisation.

Examples:
- Hackers attempting unauthorized access
- Malware or ransomware
- Insider threats
- Natural disasters affecting systems


### Vulnerability
A vulnerability is a weakness in a system that can be exploited by a threat.

Examples:
- Weak passwords
- Outdated software
- Misconfigured systems
- Poor access control


## Relationship Between Risk, Threat, and Vulnerability

- A **threat** exploits a **vulnerability**
- This leads to a **risk**
- If the risk is realized, it becomes a **security incident**


## Processes in Information Security

Information Security is a continuous lifecycle made up of the following stages:

### 1. Risk Assessment
Identifying and analysing potential risks to systems and data.

### 2. Security Planning
Designing strategies and policies to reduce or eliminate identified risks.

### 3. Implementation of Security Controls
Applying technical and administrative controls such as firewalls, authentication, and encryption.

### 4. Monitoring and Detection
Continuously observing systems to identify suspicious activity or breaches.

### 5. Incident Response
Taking immediate action when a security incident is detected using predefined response plans.

### 6. Disaster Recovery
Restoring systems and data after a major failure or attack to ensure business continuity.

### 7. Continuous Improvement
Reviewing past incidents and improving security policies, systems, and controls.


## Summary of the Security Lifecycle

Security is not a one-time task. It is a continuous cycle:

> Identify what can go wrong → Prevent attacks → Detect attacks → Respond to incidents → Recover systems → Improve defenses


## Purposes of Information Security

- Protect sensitive data from unauthorized access
- Ensure business continuity
- Maintain regulatory compliance
- Preserve brand reputation
- Safeguard intellectual property
- Enable secure digital transformation


## Common Information Security Tools

### Firewall
Controls incoming and outgoing network traffic based on security rules.

### IDS / IPS (Intrusion Detection / Prevention Systems)
- IDS: Detects suspicious activity
- IPS: Detects and blocks malicious activity

### SIEM Systems
Collect and analyse security logs from multiple sources to detect threats.

### Vulnerability Scanners
Identify weaknesses in systems, applications, and networks.

### Penetration Testing Tools
Simulate cyberattacks to identify exploitable vulnerabilities.

### Encryption Tools
Protect data confidentiality and integrity by converting it into unreadable formats without proper keys.

### Access Control Systems
Manage authentication and authorization of users.

### Security Awareness Training Platforms
Help users understand and avoid common security threats like phishing and social engineering.


## Key Takeaway
Information Security is a structured process of managing risk through prevention, detection, response, and continuous improvement.
