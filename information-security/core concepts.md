# Core Concepts in Information Security

## Overview
Information Security is built on a set of core concepts that define how systems protect data, manage access, and ensure trust. These principles form the foundation of all cybersecurity practices.


## Confidentiality
Confidentiality ensures that information is accessible only to authorized users.

### Purpose
- Prevent unauthorized access or disclosure of sensitive data

### How it is achieved
- Encryption (AES, RSA)
- Access control systems
- Authentication mechanisms (passwords, MFA)

## Integrity
Integrity ensures that data remains accurate, complete, and unaltered unless modified by authorized users.

### Purpose
- Prevent unauthorized modification or corruption of data

### How it is achieved
- Hashing algorithms (SHA-256)
- Digital signatures
- Version control systems


## Availability
Availability ensures that systems and data are accessible to authorized users when needed.

### Purpose
- Prevent downtime or disruption of services

### How it is achieved
- Backups and redundancy
- Load balancing
- Disaster recovery systems


## Authentication
Authentication verifies the identity of a user, system, or device.

### Purpose
- Ensure only legitimate entities gain access to systems

### How it is achieved
- Passwords  
- Biometrics (fingerprint, face ID)  
- Multi-factor authentication (MFA)  


## Authorization
Authorization determines what an authenticated user is allowed to do.

### Purpose
- Control access levels and permissions

### Example
- Admin can delete users, but normal users cannot


## Non-Repudiation
Non-repudiation ensures that a user cannot deny performing an action or sending data.

### Purpose
- Provide proof of origin and accountability

### How it is achieved
- Digital signatures  
- Audit logs  
- Transaction records  


## Privacy
Privacy focuses on the proper handling of personal and sensitive data.

### Purpose
- Ensure data is collected, stored, and processed responsibly

### How it is achieved
- Data minimization  
- Consent management  
- Data protection regulations (GDPR principles)  


## Summary
These core concepts define how secure systems are designed:

- Confidentiality → Who can access data  
- Integrity → Is the data correct  
- Availability → Can the system be accessed  
- Authentication → Who are you  
- Authorization → What can you do  
- Non-repudiation → Can actions be proven  
- Privacy → Is personal data protected
