# Security Principles in Information Security

## Overview
Security principles are fundamental guidelines used to design and maintain secure systems. They help reduce risk, prevent abuse, and ensure systems operate safely and reliably.


## Least Privilege
Users and systems should only have the minimum access required to perform their tasks.

### Purpose
- Reduces potential damage from compromised accounts or mistakes

### Example
- A student account should not have admin access to system servers



## Defense in Depth
Security should be implemented in multiple layers so that if one layer fails, others still provide protection.

### Purpose
- Prevents a single point of failure in security

### Example
- Firewall + antivirus + MFA + encryption working together


## Fail-Safe Defaults
Systems should deny access by default unless explicitly allowed.

### Purpose
- Prevents unauthorized access by default configuration

### Example
- New users have no permissions until granted access


## Separation of Duties
Critical tasks should be divided among multiple people or systems.

### Purpose
- Reduces risk of fraud or misuse of power

### Example
- One person creates a payment request, another approves it


## Security Through Transparency (Not Obscurity)
Systems should remain secure even if their design is known.

### Purpose
- Security should rely on strong design, not secrecy

### Example
- Encryption algorithms remain secure even when publicly known

## Summary
Security principles guide how systems are designed to minimize risk and ensure resilience against attacks.
